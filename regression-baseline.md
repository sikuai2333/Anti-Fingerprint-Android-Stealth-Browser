# 检测站点回归基线（MVP）

## 目标
为每次策略变更提供最小可执行回归标准，避免“隐蔽性提升但可用性下降”。

## 基线指标
- 页面可加载率（主站）
- 登录成功率（主站）
- 注入成功率（来自 `SpoofMetrics`）
- SSL 错误拦截是否生效
- 代理连接成功率（SOCKS5 / HTTP / VLESS）

## 回归检查步骤（发布前）
1. 启动 App，连接一个可用代理并记录连接成功/失败。
2. 打开 `https://web.max.ru`，确认首屏可加载。
3. 在设置页日志页确认 `SpoofMetrics`：
   - Attempts > 0
   - SuccessRate > 95%（测试样本 >= 20 次）
4. 访问一个证书异常测试站，确认被拦截（不能自动放行）。
5. 在启用追踪拦截时访问包含已知 tracker 的页面，确认 tracker 请求被拦截。

## 通过标准
- 主站可加载率 >= 99%
- 登录成功率 >= 98%
- 注入成功率 >= 95%
- SSL 异常拦截 100%
- 代理连接成功率 >= 95%
